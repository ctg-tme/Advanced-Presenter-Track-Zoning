Key

  • Triangle: start
  • Circle: middle points
  • Square: end

Workflow

  • Upload a 1920x1080 P60/PTZ4K camera snapshot.
    • This can be captured in the Call UI of the Codec's WebUI if the RemoteMonitoring Key is installed. Right click on the Camera image and save to file
    • Or this can done by placing the codec in a Call and take a 16:9 aspect ratio screenshot.
  • Left click on the Zone Editor canvas to create a new Point
  • 3 Points are required to form a valid TriggerZone
  • Right Click on a Point to Delete a Point
  • Right Click on a Line to Create a Point in that segment of the line
  • Drag points to adjust; use Undo and Redo for history.
  • Leverage the Tools Above to optimized your work
  • When finished, Copy the coordinates above and apply them to xConfiguration Cameras PresenterTrack TriggerZone

Notes

  • Coordinates are limited to 1920 x 1080.
  • Only 1 TriggerZone can be created for Digital Presenter Track.
  • This solution does NOT apply to Motorized Presenter Track for the Cisco Room Vision PTZ Camera. The Cisco Room Navigator offers a UserInterface to configure Trigger and Stage Zones.
  • Closing the shape is not required, the solution assume the first and last point form the shape
  • Avoid intersecting lines, as this can cause issues
  • It's best to avoid Photos and Displays within your custom TriggerZone
  • Mobile editing is optimized in landscape orientation
  • For more information, refer to the GitHub README

Help Guide

Compatibility

Build Cisco Digital Presenter Track TriggerZone coordinates for Cisco P60 and Cisco PTZ4K cameras. This workflow does not apply to Motorized Presenter Track.

What's this tool for?

Cisco's Presenter Camera Mode for the P60 and PTZ4K cameras can be configured with a custom TriggerZone shape.

By default, the WebUI of the codec only allows for a basic rectangular shape, but this is not a limitation of the API. Setting custom shapes can be done by manually configuring the coordinates of the TriggerZone and this tool was built to help you visualize and generate the coordinates for your custom TriggerZone.

The generated coordinates can be copied and pasted into xConfiguration Cameras PresenterTrack TriggerZone .

Custom TriggerZones allow you to avoid areas that could cause Camera Automation to start unexpectedly, such as a large display with people or a photo on the wall. You can also use custom TriggerZones to avoid areas that are not relevant to your presentation, such as a podium or a table in front of the presenter.

Placement Helpers

Snap to Grid is enabled by default and shows a placement preview. Adjust Grid spacing when you need coarser or finer alignment. On mobile, landscape orientation gives the most room for drawing.

Coordinates

Type or paste x,y coordinate pairs. Complete pairs apply automatically; incomplete pairs are ignored until finished and are highlighted without changing your text.

Shortcuts

  • Cmd/Ctrl+C copies valid coordinates.
  • Cmd/Ctrl+V pastes coordinates into the editor.
  • Cmd/Ctrl+Z undoes the last zone action.
  • Cmd/Ctrl+Shift+Z or Cmd/Ctrl+Y redoes it.

Feedback

Bugs are easiest to fix from GitHub Issues where screenshots and repro steps can be attached.
